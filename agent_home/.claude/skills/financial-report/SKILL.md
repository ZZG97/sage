# Financial Report Analysis Skill

用户提供上市公司财报（PDF/图片/文本/网址），按结构化框架分析，输出报告文件。

## 输入方式

用户可能通过以下方式提供财报：
1. **PDF 文件** — 直接读取
2. **图片** — 截图/拍照的财报页面，直接读取
3. **文本** — 粘贴的财报数据
4. **网址** — 用 WebFetch 或 Playwright 抓取财报内容

## 执行流程

### Step 1: 获取财报数据

**如果是网址（东方财富、港交所、巨潮等财经网站）：**
> ⚠️ 这些网站大量使用 JavaScript 渲染，WebFetch 通常只能拿到 JS 变量，没有实际内容。
> **推荐顺序：**
> 1. 直接用 Playwright `browser_navigate` 打开 URL，从页面 title 和 URL 确认公司+报告类型
> 2. 从 Playwright snapshot 中提取 PDF 附件链接（通常在"下载附件"按钮附近）
> 3. 用 `curl -L -o` 下载 PDF 到 `workspace/uploads/`
> 4. 用 `pdftotext` 提取文本（需要 `brew install poppler`）

**如果是 PDF 文件：**
> ⚠️ Read 工具的 PDF 支持依赖 `pdftoppm`，如果报错"No PDF renderer"，说明没有该工具。
> **推荐方案（绕过 Read 工具，直接用命令行）：**
> 1. 用 `Bash` 执行 `pdftotext <file_path> -` 提取纯文本
> 2. 用 `grep -n "关键词"` 定位财务报表明细所在行号
> 3. 用 `sed -n 'start,end p'` 截取目标段落

**关键文本定位技巧（以港股年报为例）：**
```
# 定位各报表位置
grep -n "合併損益表\|合併資產負債表\|合併現金流量表" <file>

# 定位特定科目
grep -n "經營活動所得現金淨額\|總收入\|毛利\|淨利潤" <file>

# 提取收入/利润区域（通常是报表正文开始位置）
sed -n '340,470p'   # 损益表
sed -n '2621,2750p' # 现金流量表
```

**如果是文本：** 直接使用。

**如果数据不完整：** 在报告中注明"数据缺失"，不编造数据。

### Step 2: 识别基础信息
- 报告类型：季报/半年报/年报
- 报告期间
- 公司名称、股票代码、主营业务、行业

### Step 3: 按七步框架分析

#### 3.1 核心财务指标
- **营收**：当期数值、同比增长率、多年趋势（如有）、复合增长率、营收结构（产品/地区）
- **净利润**：当期数值、同比增长率、净利润率、ROE、ROA

#### 3.2 现金流分析
- 经营性现金流及与净利润比值
- 自由现金流 = 经营性现金流 - 资本支出
- 现金流趋势和异常原因

#### 3.3 费用结构
- 销售/管理/研发/财务费用：金额、占营收比、同比变化
- 费用增长 vs 营收增长对比

#### 3.4 存货风险（制造业重点）
- 存货余额、占总资产比、周转率
- 存货结构和减值准备

#### 3.5 异常指标识别
- 波动超过 30% 的指标逐项分析
- 应收账款、预付账款、短期借款等异常变动

#### 3.6 关键财务比率计算

逐项计算，列出公式和结果，并与上期对比：

**盈利能力指标**
- 毛利率 = (营收 - 营业成本) / 营收 × 100%
- 净利率 = 净利润 / 营收 × 100%
- ROE = 净利润 / 股东权益 × 100%
- ROA = 净利润 / 总资产 × 100%

**运营效率指标**
- 存货周转率 = 营业成本 / 平均存货（次/年）
- 应收账款周转率 = 营收 / 平均应收账款（次/年）
- 总资产周转率 = 营收 / 平均总资产（次/年）

**财务风险指标**
- 资产负债率 = 总负债 / 总资产 × 100%
- 流动比率 = 流动资产 / 流动负债
- 速动比率 = (流动资产 - 存货) / 流动负债

**现金流指标**
- 经营性现金流/净利润比 = 经营活动现金流净额 / 净利润 × 100%
- 经营性现金流/营收比 = 经营活动现金流净额 / 营收 × 100%
- 自由现金流 = 经营活动现金流净额 - 资本支出

如有多年数据，逐年列表对比趋势。

#### 3.7 综合评估
- 财务健康度：优秀/良好/一般/较差
- 主要风险点（3-5 个）
- 成长性、盈利能力、财务稳定性评估
- 投资建议及理由

### Step 4: 风险预警

**红色预警** 🔴（需要重点关注）：
- 营收同比增长 < 0%
- 营收增长与行业平均偏差 > 20%
- 净利润率同比下降 > 30%
- ROE < 行业平均的 50%
- 经营性现金流为负
- 净利润与经营性现金流严重不匹配（差异 > 50%）— 可能存在利润操纵
- 费用率同比上升 > 20%
- 销售费用/营收比 > 行业平均 + 50%
- 存货周转率同比下降 > 30%
- 存货占总资产比 > 行业平均 + 50%
- 资产负债率 > 70%
- 流动比率 < 1.0

**黄色预警** 🟡（需要持续监控）：
- 营收增长放缓（低于行业平均）
- 毛利率持续下降
- 费用增长快于营收增长
- 应收账款占比过高或增速异常
- 预付账款大幅增加
- 短期借款大幅变化
- 关联交易金额异常

对每个触发的预警，给出具体数据和原因分析。

### Step 5: 输出报告

报告保存到 `agent_home/workspace/outputs/financial/` 目录，文件名格式：
`{公司名称}_{报告类型}_{报告期间}_分析报告.md`

示例：`比亚迪_年报_2024_分析报告.md`

## 报告类型差异化分析

**季报分析要点**（在报告中额外关注）：
- 季节性调整：识别行业季节性特征，与去年同期同季度对比
- 环比增长趋势分析
- 累计业绩进度（Q3 时看前三季度完成度）
- 下季度业绩预告和订单/合同执行情况

**半年报分析要点**：
- 上半年 vs 去年上半年同比
- 全年业绩预估（线性外推 + 季节性修正）
- 中期分红政策变化

**年报分析要点**（在报告中额外关注）：
- 审计报告类型和意见（标准无保留 / 保留 / 否定 / 无法表示）
- 重要会计政策变化
- 经营计划完成情况 vs 年初目标
- 核心竞争力分析和发展战略执行情况
- 下一年度经营计划和资本支出计划
- 分红方案

## 常见陷阱与经验

1. **WebFetch 拿不到财经网站内容** — 东方财富、港交所、巨潮等几乎全站 JS 渲染，WebFetch 只返回空壳。不要在 WebFetch 上浪费时间，直接 Playwright + curl。

2. **Playwright 无法直接读取 PDF 文本** — 导航到 PDF URL 会得到空白页。正确做法：用 curl 下载到本地，再用 pdftotext 命令行提取。

3. **pdftotext 中文竖排/合并单元格格式混乱** — 提取出来是纯文本，数字和标签可能不在同一行。需要用 grep 配合关键词定位，再用 sed 截取段落。

4. **年报三张表不一定连续** — 损益表、资产负债表、现金流量表通常在不同页。资产负债表数据有时在 PDF 中格式最乱，难以精确提取。如数据缺失，用估算或标注"数据缺失"。

5. **注意金额单位** — 港股年报常用"人民币千元"（¥000），A股常用"万元"或"元"，美股用当地货币。计算前先确认单位，避免数量级错误。

6. **存货跌价准备要同时看计提和转出** — 评估存货质量时，不能只看期末余额或计提数，还需关注"本期计提"和"本期转出"（出售后拨回）两个发生额，结合起来才能判断存货实际损耗情况。

## 注意事项
- 所有结论必须有数据支撑，不做无依据推测
- 如果财报数据不完整，在报告中注明"数据缺失"而非编造数据
- 分析语言跟随用户语言（中文财报用中文分析）
- 报告末尾附免责声明：本报告由 AI 生成，仅供参考，不构成投资建议
